# Modules in Frappe Framework

A DocType will always belong to a module, to enable easier grouping of related models. Frappe comes with a lot of built-in modules. For example:

1. **Core** - contains doctypes like DocType, DocField, Report, System Settings, etc.
2. **Desk** - contains doctypes like ToDo, Event, Note, Kanban Board, etc.
3. **Email** - contains doctypes like Email Account, Newsletter, Email Group, etc.

## Directory Structure

Modules also help in grouping code files in directories. The controller files generated by a DocType will live in its respective Module directory.

```
frappe
├── commands
├── config
├── core
│   ├── doctype
│   │   ├── doctype
│   │   ├── docfield
│   │   ├── report
│   │   ├── system_settings
│   │   ├── ...
│   │   └── ...
│   ├── page
│   ├── report
│   └── web_form
...
...
├── desk
│   ├── doctype
│   │   ├── ...
│   │   ├── event
│   │   ├── kanban_board
│   │   ├── note
│   │   └── todo
│   ├── form
│   ├── page
```

When you create a new app, a module with the same name as the app is automatically created. You can create more modules within your app as needed.

## Custom Modules

When building applications on Frappe, you should create custom modules to organize your DocTypes and other components. This ensures better organization and maintainability of your codebase.
