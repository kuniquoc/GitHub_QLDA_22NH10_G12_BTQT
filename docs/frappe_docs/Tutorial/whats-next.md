# What's Next

## Congratulations!

You have successfully completed the Frappe Framework tutorial series! Throughout this journey, you've built a complete Library Management System with various features:

- Created multiple DocTypes to model entities and transactions
- Implemented server-side business logic using controller methods
- Enhanced the user experience with client-side form scripts
- Created web views for public access

But this is just the beginning of what you can accomplish with Frappe Framework. Let's look at where you can go from here.

## Exploring Advanced Topics

To deepen your understanding of Frappe Framework, we recommend exploring these advanced topics:

1. **Architecture**: Understand the underlying architecture of Frappe Framework, including how the ORM works, the request-response cycle, and the event system.

2. **DocType Advanced Features**: Explore advanced DocType features such as virtual fields, dynamic links, document states, and workflows.

3. **Document API**: Learn more about the Document API to manipulate documents programmatically, including filtering, sorting, and aggregation operations.

4. **Form API**: Dive deeper into the client-side Form API to create more interactive and dynamic interfaces.

5. **Bench CLI**: Master the Bench command-line interface to automate common development tasks and manage your Frappe applications efficiently.

6. **REST API**: Learn how to build RESTful APIs to integrate your Frappe application with other systems.

7. **Scheduled Jobs**: Implement background tasks and scheduled jobs using Frappe's job queue system.

8. **Custom Reports**: Create custom reports and dashboards to analyze your application data.

9. **Deployment**: Learn about different deployment strategies for your Frappe application in production environments.

## Building Your Own Applications

Now that you have a solid foundation in Frappe Framework, you can start building your own applications:

1. **Plan your data model**: Design your DocTypes carefully, considering the relationships between different entities.

2. **Implement business logic**: Use controller methods to implement complex validations and workflows.

3. **Create intuitive interfaces**: Design user-friendly forms and list views.

4. **Build public web pages**: Create web views for public users.

5. **Test thoroughly**: Write tests for your application to ensure it works as expected.

6. **Deploy to production**: Set up your application in a production environment.

## Contributing to Frappe
